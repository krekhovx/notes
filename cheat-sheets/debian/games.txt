--- game-data-packager
$ apt-get install game-data-packager

game-data-packager — это утилита в Debian, которая помогает легально
установливать старые коммерческие игры из оригинальных файлов
(CD, GOG, Steam и т.д.) на современных Linux-системах.

Зачем нужен:
Многие классические игры (Doom, Quake, Heretic, Hexen, Unreal и другие) давно
имеют открытые движки, но их игровые данные (музыка, уровни, текстуры)
по-прежнему распространяются только на платных дисках или в цифровых магазинах.
Debian не может включить эти данные в свои репозитории из-за авторских прав.

Как работает:

- Пользователь указывает исходники оригинальной игры (например, указывает путь к
  CD или скачанным файлам).

- `game-data-packager` автоматически собирает Debian-пакет с игровыми данными,
  который потом можно установить как обычный пакет.

- После этого можно запускать игру через свободный движок (например, через
  `prboom` для Doom).

Для чего это удобно:
- Легко интегрируется с системой (игры появляются в меню).
- Устанавливает только нужные данные.
- Не нарушает лицензии.

Пример использования:

$ game-data-packager --help # список поддерживаемых игр и опций
$ game-data-packager -i quake id1/*.pak # установка генерируемого пакета

$ game-data-packager -n quake id1/*.pak # не устанавливать генерируемый пакет
$ apt-get install -y ./*.deb

Вывод:
`game-data-packager` — удобный способ легально добавить игровые данные для
старых игр в Debian-систему, если у вас есть оригинальные файлы.


--- play.it
play.it — это скриптовый инструмент, который помогает устанавливать и запускать
коммерческие (и не только) игры на Linux, включая Debian, создавая из
установочных файлов игры удобные deb-пакеты.

$ apt-get install play.it

play.it использует установщик игры (например, .exe из GOG), распаковывает его
содержимое и создает deb-пакет с нужными файлами и настройками для запуска игры
через Wine или нативно (если поддерживается). Затем нужно установить этот
deb-пакет, и игра появится меню приложений и будет работать через Wine (или
другой совместимый слой).

$ play.it setup_game.exe
$ apt-get install -y game.deb

# список поддерживаемых игр и опций
$ play.it --list-available-scripts

# скрипты для игр хранятся:
/usr/share/games/play.it/


--- Игровые порты
Игровые порты — это не официальные версии игр, а адаптированные или заново
написанные движки старых игр, которые позволяют запускать их на современных
системах. Они жизненно важны для сохранения игровой истории и ретро-гейминга,
часто дают игре "вторую жизнь".

> Примеры игровых портов:

ioquake3 — порт движка Quake III Arena. Исходный код был открыт id Software,
энтузиасты улучшили его, добавили поддержку новых ОС и расширили возможности.

VCMI — попытка воссоздать движок Heroes of Might and Magic III. Исходников
оригинала нет, всё пишется с нуля по поведению оригинала.

DevilutionX — реверс-инженеренный движок Diablo 1. Исходника не было, энтузиасты
восстановили код по бинарнику.

> Как пишутся игровые порты:

Иногда с исходников, если их выложил разработчик (как Doom, Quake).

Чаще всего — с помощью реверс-инжиниринга: смотрят, как работает оригинальная
программа, анализируют данные, изучают бинарники (exe-файлы), и воссоздают код
на новом языке.

> Кто этим занимается:

Открытое сообщество энтузиастов, часто фанаты игры или программисты, которым
интересен вызов.

Иногда — одиночки, чаще — небольшие команды.

> Цели портов:

Позволить запускать любимые игры на современных ОС, где они иначе не работают.

Исправить баги, добавить поддержку современных разрешений, онлайн-игры, новых
контроллеров и т.д.

Иногда — добавить новые возможности (моды, мультиплеер, графические улучшения).
